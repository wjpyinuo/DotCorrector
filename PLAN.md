# DotCorrector 开发计划

## 三级纠错流水线

### 第一遍：错别字词典（本地）
- 使用常见错别字对照表（同音字、形近字）
- 逐词匹配替换，极快、零成本
- 示例：的/地/得、在/再、已/以、做/作、象/像
- 词典来源：开源中文错别字词典（如 `common_typos.txt`）

### 第二遍：百度 pycorrector（本地）
- 基于统计模型 + 规则的开源纠错引擎
- 处理上下文相关的语法错误
- 无需联网，速度快
- 依赖：`pip install pycorrector`

### 第三遍：大模型 AI 精校（API）
- 调用 MiMo / DeepSeek API，做语义级纠错和润色
- 能纠正在/再、的/地/得 等依赖上下文的错误
- 大文件分段发送，避免超 token 限制
- API Key 在设置页配置

---

## 纠错增强功能

### 1. 改动标记（Diff）
- 每一遍独立记录改动位置和内容
- 生成逐条改动列表，方便回溯
- 支持"只看某一遍的改动"

### 2. 跳过机制
- 设置页中可选择启用哪些遍（词典/pycorrector/大模型）
- 至少保留一遍，不允许全部跳过
- 关闭大模型时自动跳过第三遍

### 3. 分段处理
- 大文件（xlsx/pptx/长文档）自动按段落/单元格分块
- 大模型遍按块发送，每块独立纠错
- 保持原始顺序，合并结果

### 4. 纠错报告
- 处理完成后生成报告，包含：
  - 总共修改了 N 处
  - 每遍分别修改了多少处
  - 改动详情列表（原文 → 修正后）
- 可导出为文本文件

## 预览功能

### 改前/改后对比
- 纠错完成后进入预览页面
- 左右分栏或上下对照展示原文与修正版
- 支持逐条查看和整体浏览

### 高亮显示
- 修改处用颜色高亮标注（如黄色/绿色背景）
- 不同遍的改动用不同颜色区分：
  - 词典纠错 → 蓝色
  - pycorrector → 橙色
  - AI 精校 → 绿色
- hover/点击高亮处显示修改详情（原文 → 新文）

### 逐条审核
- 每条修改可单独接受/拒绝
- "全部接受" / "全部拒绝" 快捷操作
- 接受/拒绝后实时更新预览

### 导出
- 最终版导出为原格式文件（docx/xlsx/pptx/txt）
- 保留原始排版样式
